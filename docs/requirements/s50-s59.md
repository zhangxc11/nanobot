# 需求 §50–§59

## §50 Inject 队列 Drain — 防止消息丢失

**状态**: ✅ 已完成 (Phase 47)

### 背景

当多条消息（subagent 结果 + 用户 inject）在同一个 tool 执行期间到达 inject 队列时，
`check_user_input()` 每次 checkpoint 只消费一条，导致后续消息被静默丢弃。

### 需求

1. **Tool-call checkpoint drain**：每次 inject checkpoint 必须循环消费队列中所有 pending 消息
2. **Final-response checkpoint**：当 LLM 返回纯文本（无 tool_calls）时，在退出循环前检查队列；
   如果有 pending 消息，持久化后 `continue` 让 LLM 继续处理
3. **日志全量记录**：webserver/worker 的 inject 和消息日志不再截断

### 验收标准

- [x] 多条消息同时到达时全部被消费（test_multiple_messages_drained_after_tool）
- [x] 单条消息仍正常工作（test_single_inject_still_works）
- [x] 最终响应期间到达的消息触发 LLM 继续处理（test_inject_during_final_response_triggers_continuation）
- [x] 无 inject 时正常退出（test_no_inject_on_final_response_exits_normally）
- [x] dict + string 混合 inject 类型均正确处理（test_mixed_dict_and_string_inject）
