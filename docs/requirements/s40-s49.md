# 需求归档 §40–§49

## §40 SubagentManager 单例化 + 跨进程 follow_up 恢复

### 背景

Web worker 模式下，每次请求创建新的 `_create_runner() → AgentLoop → SubagentManager`，turn 结束后被 GC。导致：

- **跨 turn**：同一 session 的后续请求无法找到之前 spawn 的 subagent（`status`/`follow_up`/`stop`/`list` 返回 "Unknown subagent task_id"）
- **跨进程**：worker 进程重启后，即使 subagent session 文件还在磁盘上，也无法通过 task_id 找到对应的 session

Gateway 模式不受影响（AgentLoop 是进程级长生命周期单例）。

### 设计方案

两层解决：进程内单例化 + 跨进程按规则恢复。

#### 子需求 1：AgentLoop 接受外部 SubagentManager

**改动文件**：`nanobot/agent/loop.py`

`AgentLoop.__init__` 新增可选参数 `subagent_manager: SubagentManager | None = None`：
- 传入 → 使用外部实例（web worker 单例模式）
- 不传（默认 None）→ 自己创建（gateway 模式，保持现有行为不变）

```python
class AgentLoop:
    def __init__(
        self,
        ...
        subagent_manager: "SubagentManager | None" = None,  # §40 新增
    ):
        ...
        if subagent_manager is not None:
            self.subagents = subagent_manager
        else:
            self.subagents = SubagentManager(
                provider=provider,
                workspace=workspace,
                bus=bus,
                ...  # 现有参数不变
            )
```

**注意**：`AgentRunner` 也需要透传此参数（如果 AgentRunner 封装了 AgentLoop 构造）。

#### 子需求 2：web-chat worker 维护 SubagentManager 单例

**改动文件**：`web-chat/worker.py`

在模块级维护 SubagentManager 单例，所有 `_create_runner()` 共享：

```python
_subagent_manager: SubagentManager | None = None
_subagent_manager_lock = threading.Lock()

def _get_subagent_manager() -> SubagentManager:
    """获取或创建 SubagentManager 单例。"""
    global _subagent_manager
    if _subagent_manager is not None:
        return _subagent_manager
    with _subagent_manager_lock:
        if _subagent_manager is not None:
            return _subagent_manager
        # 从配置构建（参数与 _create_runner 中 AgentLoop 的一致）
        _subagent_manager = SubagentManager(...)
        return _subagent_manager

def _create_runner():
    agent_loop = AgentLoop(
        ...
        subagent_manager=_get_subagent_manager(),  # §40: 共享单例
    )
```

**注意事项**：
- SubagentManager 持有的 `provider`（ProviderPool）已经是 worker.py 的模块级单例，可直接复用
- `SessionManager` 实例需要共享（或确认无状态可共享）
- `task_keeper` 回调已有模块级 `_keep_subagent_task()` 函数，可直接复用
- `session_messenger` 需要确认是否可作为单例（WorkerSessionMessenger 是无状态 HTTP 调用，可以）

#### 子需求 3：跨进程恢复 — `_check_ownership` 按规则重建 meta

**改动文件**：`nanobot/agent/subagent.py`

当 `_check_ownership()` 在内存 `_task_meta` 中找不到 task_id 时，利用 subagent session 的**确定性命名规则**尝试恢复：

```
task_id = "a1b2c3d4"
parent_session_key = "webchat:1773141981"
→ subagent_session_key = "subagent:webchat_1773141981_a1b2c3d4"
→ session 文件路径 = sessions/subagent_webchat_1773141981_a1b2c3d4.jsonl
```

**不需要扫描磁盘**，直接按规则构造 session key，检查文件是否存在即可（O(1) 文件 stat）。

```python
def _check_ownership(self, parent_session_key: str, target_task_id: str) -> SubagentMeta:
    meta = self._task_meta.get(target_task_id)
    if meta is None:
        # §40: 尝试从磁盘恢复
        meta = self._recover_meta(target_task_id, parent_session_key)
    if meta is None:
        raise ValueError(f"Unknown subagent task_id: {target_task_id}")
    if meta.parent_session_key != parent_session_key:
        raise ValueError(f"Subagent {target_task_id} does not belong to this session")
    return meta

def _recover_meta(self, task_id: str, parent_session_key: str) -> SubagentMeta | None:
    """按命名规则重建 SubagentMeta，用于跨进程 follow_up 恢复。
    
    只恢复 follow_up 所需的最小信息（task_id ↔ session_key 映射），
    不恢复 status/iteration 等运行时状态（进程重启后无意义）。
    """
    if not self.session_manager:
        return None
    
    parent_sanitized = parent_session_key.replace(":", "_")
    subagent_key = f"subagent:{parent_sanitized}_{task_id}"
    
    # 检查 session 文件是否存在
    session_path = self.workspace / "sessions" / f"subagent_{parent_sanitized}_{task_id}.jsonl"
    if not session_path.exists():
        return None
    
    meta = SubagentMeta(
        task_id=task_id,
        subagent_session_key=subagent_key,
        parent_session_key=parent_session_key,
        label="(recovered)",
        origin={"channel": "unknown", "chat_id": "unknown"},
        status="unknown",
        persist=True,
    )
    
    # 缓存到内存
    self._task_meta[task_id] = meta
    self._session_tasks.setdefault(parent_session_key, set()).add(task_id)
    
    return meta
```

#### 子需求 4：`list_subagents` 跨进程增强

`list_subagents()` 需要列出某个 parent session 的所有 subagent。跨进程后内存为空，需要从磁盘补充。

使用 glob 匹配该 parent 前缀的 session 文件（通常只有几个文件，很轻量）：

```python
def list_subagents(self, parent_session_key: str) -> str:
    # §40: 先从磁盘补充
    self._load_disk_subagents(parent_session_key)
    # 然后走原有逻辑...

def _load_disk_subagents(self, parent_session_key: str) -> None:
    """从磁盘 session 文件补充该 parent 的 subagent meta 到内存。"""
    parent_sanitized = parent_session_key.replace(":", "_")
    prefix = f"subagent_{parent_sanitized}_"
    sessions_dir = self.workspace / "sessions"
    if not sessions_dir.exists():
        return
    
    for path in sessions_dir.glob(f"{prefix}*.jsonl"):
        task_id = path.stem.rsplit("_", 1)[-1]
        if task_id not in self._task_meta:
            self._recover_meta(task_id, parent_session_key)
```

### 行为总结

| 场景 | 机制 | 开销 |
|------|------|------|
| 进程内跨 turn（status/follow_up/stop） | SubagentManager 单例，内存中直接查 | O(1) |
| 跨进程 follow_up/status/stop | `_check_ownership` 按规则构造 + 验证文件存在 | O(1) 文件 stat |
| 跨进程 list | glob 匹配该 parent 前缀的 session 文件 | O(n)，n = 该 parent 的 subagent 数 |

### 影响范围

| 文件 | 改动 | 风险 |
|------|------|------|
| `nanobot/agent/subagent.py` | `_check_ownership` 加 fallback；新增 `_recover_meta` + `_load_disk_subagents` | 低 |
| `nanobot/agent/loop.py` | `__init__` 新增 `subagent_manager` 可选参数 | 低 |
| `web-chat/worker.py` | SubagentManager 单例化 | 中 |
| Gateway 模式 | 无改动 | 无 |

### 不做的事

- ❌ 不持久化 status/iteration/last_tool 等运行时信息到磁盘
- ❌ 不改变 session 文件格式或命名规则
- ❌ 不影响 gateway 模式的任何行为

### 测试要点

1. **`_recover_meta` 单元测试**：
   - session 文件存在 → 成功恢复 meta，缓存到内存
   - session 文件不存在 → 返回 None
   - session_manager 为 None → 返回 None
   - 恢复后的 meta 的 parent_session_key 正确
   - 重复调用不重复创建（从缓存返回）

2. **`_load_disk_subagents` 单元测试**：
   - 磁盘有多个 subagent session 文件 → 全部加载
   - 已在内存中的不重复加载
   - sessions 目录不存在 → 不报错

3. **`_check_ownership` 增强测试**：
   - 内存中有 → 直接返回（现有行为不变）
   - 内存中没有，磁盘有 → 恢复并返回
   - 内存中没有，磁盘也没有 → 抛 ValueError

4. **follow_up 跨进程恢复集成测试**：
   - 创建 SubagentManager A → spawn subagent → 记录 task_id
   - 创建 SubagentManager B（模拟进程重启）→ follow_up 同一个 task_id → 成功恢复并发送消息

5. **AgentLoop subagent_manager 参数测试**：
   - 传入外部 SubagentManager → 使用传入的实例
   - 不传 → 自己创建（默认行为）

6. **回归测试**：现有 spawn/follow_up/stop/status 测试全部通过
